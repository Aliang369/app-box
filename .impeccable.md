## Design Context

### Users
游戏应用盒子用户，主要是手游玩家，在碎片化时间浏览和发现新游戏。使用场景是快速浏览、搜索、收藏和下载游戏。

### Brand Personality
纯净、克制、功能至上。三个关键词：**white / minimal / functional**。

### Aesthetic Direction
- **纯白底色**：背景使用纯白 #FFFFFF，不使用渐变、不使用深色
- **精简克制**：去掉所有装饰性元素（模糊光斑、渐变色、玻璃效果、厚重阴影），只保留功能必需的视觉元素
- **无渐变色**：所有颜色使用纯色，不使用任何 linear-gradient 或 radial-gradient
- **功能优先**：每个元素都服务于功能，搜索框就是搜索框，卡片就是信息载体
- **极细分割**：使用 1px 浅灰线条分割区域，不用色块
- **文字层级**：通过字号和字重拉开层级，不依赖颜色
- **图标精简**：使用 lucide 细线条图标，统一灰色调

### Color Palette
- 背景：#FFFFFF（纯白）
- 一级文字：#111827（近黑）
- 二级文字：#6B7280（中灰）
- 三级文字：#9CA3AF（浅灰）
- 分割线：#F3F4F6（极浅灰）
- 主色调：#111827（黑色，用于强调和交互态）
- 卡片背景：#FFFFFF（纯白，用边框区分）

### Design Principles
1. **减法设计**：能去掉的装饰全部去掉，只留功能
2. **白色呼吸**：大量留白，让内容自然呼吸
3. **文字即界面**：用排版而非装饰传达信息
4. **一致克制**：所有页面统一风格，不追求花哨
5. **交互明确**：点击区域清晰，反馈直接

## Mobile Header Spec

移动端二级页面顶部统一遵循以下规则：

- **统一组件**：优先复用 `packages/mobile/src/components/PageHeader.vue`，不要在页面里重复手写顶部结构。
- **统一安全区**：顶部容器必须使用 `padding-top: calc(var(--status-bar-height, 0px) + 16px)`，禁止再写 `pt-24`、`pt-20`、`pt-12`、`top-14` 这类硬编码顶部留白。
- **统一结构**：头部固定为三段式布局。左侧返回按钮，中间页面标题，右侧可选操作位；标题必须视觉居中。
- **统一视觉**：头部使用纯白背景、左右 `24px` 内边距、底部 `20px` 内边距、`1px` 浅灰底边 `border-gray-100`。返回按钮使用 `40px * 40px` 的浅灰圆形点击区。
- **统一标题**：标题使用 `text-[18px] font-black text-gray-900 tracking-tight`，只保留一个中文标题，不再叠加英文副标题。
- **统一返回逻辑**：优先 `navigateBack()`；当页面是直达打开时，必须提供 `fallback-url` 回到合理的 tab 页。
- **统一适用范围**：礼包、收藏、足迹、详情，以及后续新增的所有二级内容页，都必须沿用这套顶部规范；除首页、搜索页这类明确的特殊布局外，不得混用原生导航栏和自定义头部。
